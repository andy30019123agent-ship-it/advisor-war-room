## 🔴 壞掉（阻斷級）

### 🔴 1. 沒有基本面資料的標的（ETF／缺財報）查股票會整頁壞掉，顯示「請更新 App」
- **重現**：查股票 tab 輸入 `0050`（元大台灣 50，ETF 無 PER/財報）→分析
- **結果**：畫面只顯示「請更新 App」，完全看不到任何分析內容
- **根因（已查證）**：
  - `/api/analyze?stock=0050` 實際回傳 200，資料完整（`meta.schema_version: 1`），但 `context.timeframes.mid.stance` 欄位值是 `"缺"`
  - 前端 Zod schema `app/src/types/contract.ts:15` 的 `StanceSchema` enum 只接受 `['偏多','中性偏多','中性','中性偏空','偏空']`，不包含 `"缺"`
  - 後端 `warroom/decision_engine.py:10` 與 `warroom/primary_decision.py:331` 在資料不足（`na`）時會產出中文字串 `"缺"` 當作 stance，導致 Zod 驗證失敗 → 前端誤判為「App 版本不符」，丟出使用者完全看不懂、也無法自救的「請更新 App」訊息
  - 這不是版本不符，是資料契約沒對齊，會影響**任何缺基本面資料的標的**（不只 0050，理論上所有無 PER／財報的 ETF、新股都會中招）
- **證據**：console `[error] Failed to load resource: 404` ×2；network `GET /api/analyze?stock=0050` 200 但內容含非法 enum 值（已用 evaluate_script 讀出完整 response body 核對）
- **影響**：查股票功能對 ETF／缺資料標的完全失效，且錯誤訊息完全誤導（叫使用者去更新 App，但其實是資料格式問題，更新 App 也不會好）
- **建議**：後端 stance 缺資料時應輸出 Zod schema 允許的值（例如沿用 `"中性"`），或前端 `StanceSchema` enum 補上 `"缺"` / 改成 nullable
- **截圖**：`acceptance-2026-07-18/jointtest-07-BUG-0050-請更新App.png`

## 🟡 不順手

### 🟡 3. 每次查非快取標的都會在 console 噴一個紅字 404
- 查股票／持股 tab 只要標的不在預先產生的 `/data/stocks/<id>.json` 快取清單裡（例如 0050、8299、2317、6505、9999、abc，甚至使用者自己在持股 tab 加的任何非追蹤股），都會先打一次靜態檔案 404，再 fallback 打 `/api/analyze`。這是設計內的 fallback 機制、功能上沒壞，但 404 會被瀏覽器記成 console `[error]` 紅字，每次查詢都會出現，不符合「console 零容忍紅字」的驗收標準。
- 建議：改用 `HEAD` 探測或直接判斷是否為追蹤清單內代號，別讓 404 進 console；或前端把這條 fetch 包一層讓它不出現在 console error。

### 🟡 4. 非快取標的每次查詢，網路請求都打了兩輪（重複 fetch）
- 查 0050／8299／2317／9999／abc 時，network 面板都看到同一組請求（`/data/stocks/X.json` 404 + `/api/analyze?stock=X`）**各出現兩次**，等於同一次查詢實際打了 2 次 API（第一筆快取命中的 2330 反而只打 1 次）。
- 已確認 `app/src/main.tsx` 有包 `<StrictMode>`，怀疑跟這個有關，但無法 100% 定論（StrictMode 理論上只在 dev 模式雙渲染）；無論根因為何，實測結果就是「查一次、打兩次 API」，會白白浪費即時分析的 API 額度／算力。
- 建議工程排查（React Query queryFn 是否被重複觸發）。

### 🟡 5. 新增持股表單沒有自動帶入公司名稱
- 新增持股時「股票代號」輸入框有 `autocomplete="list"` / `haspopup="menu"` 屬性，看起來像設計了自動完成，但實測填入 2330 完全沒有跳出建議清單，「名稱」欄位還是要使用者自己手打（placeholder 是「例如 台積電」提示手動輸入）。體驗上不如預期的智慧。

### 🟡 6. 今日 tab 的「觀察清單」目前完全是空的，功能沒東西可測
- `daily.json` 裡 `watch: []`，前端邏輯（`Today.tsx:110`）在陣列長度 0 時正確隱藏整個區塊，程式碼本身沒問題，但這代表「觀察清單點擊行為」這個驗收項目目前**沒有資料可以驗證**——不是壞掉，是內容還沒填。

### 🟡 7. iOS「加入主畫面」缺少 apple-touch-icon
- 檢查 `<head>` 沒有 `<link rel="apple-touch-icon">`，也沒有 `<meta name="apple-mobile-web-app-capable">`。manifest 本身（`/manifest.webmanifest`）正常、icon（`/pwa-192.svg`, `/pwa-512.svg`）都能 200 讀到，Service Worker 也正常註冊（1 個，scope `/`，狀態 activated）。但 iOS Safari「加入主畫面」主要吃 apple-touch-icon 而非 manifest icons，沒有這個標籤，使用者在 iPhone 加到主畫面時圖示可能不是預期的 App icon（例如變成整頁截圖縮圖）。這款是天天要用的戰情室 App，圖示辨識度會直接影響 Andy 跟客戶的使用體驗。
