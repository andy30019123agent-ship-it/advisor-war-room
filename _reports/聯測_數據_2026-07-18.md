## 4. API 即時性：`/api/analyze?stock=1301`

- 回應：`price.close = 62.8`
- TWSE MIS `tse_1301.tw`：`z=62.8000`，`d=20260717`
- **收盤價：✅ 一致**

- 但 `meta.data_date = "2026-07-18"`（呼叫當下的台北日期，週六），而**同一份回應**內 `context.lights.chips.facts` 明寫「最新日 **2026-07-17**」
  - **結果：❌ 同一 API 回應內部自相矛盾** — `meta.data_date` 應該代表「行情資料日」，卻用了呼叫當下的今天日期，跟回應內其他欄位標的行情日（07-17）對不上
  - **根因（已讀原始碼確認）**：`api/analyze.py` 第 125 行 `build_meta(sources=["FinMind REST (lite)"])` 呼叫時**沒有傳入 `data_date` 參數**；`warroom/build_snapshots.py` 第 79-83 行的函式簽名與註解寫明「`data_date`＝行情資料日（契約定義），不是產生日；抓不到行情日才 fallback 今天」，但這裡沒帶，於是落回 `_today()`（即 2026-07-18）。對照批次流程 `build_snapshots.py` 第 448 行有正確帶入 `market_inputs.get("trade_date")`，可見這是即時查詢路徑（`api/analyze.py`）獨有、可重現的欄位漏帶 bug，不是隨機噪音。
