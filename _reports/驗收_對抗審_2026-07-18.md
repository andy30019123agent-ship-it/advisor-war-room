| 查股票 2330 → 顯示分析 | ✅ pass | 首次查詢 2.4s 回傳完整分析卡（續抱／防守價 2,106.8／證據拆解 5 個分頁），`375-查股票-2330.png` |
| 快取後應快 | 🟡 **待確認** | 第二次查同一支 2330，耗時 3.4s，**沒有變快**（甚至略慢）。查 Network 發現 2330 這種「已在每日 tracked 清單」的股票，前端根本沒打 `/api/analyze`，而是直接讀 `/data/stocks/2330.json`（靜態快照），瀏覽器 HTTP cache 對 304 有生效，但兩次量到的『使用者體感時間』沒有明顯差異，不算違反但沒達到「快取後應快」的體感預期 |
| 查股票 9999 → 友善「查無資料」空狀態，非白屏 | 🔴 **fail** | 實測不是白屏（有畫面），但顯示的是通用錯誤「**分析失敗，請稍後再試**」，不是任務要求的友善「查無資料」文案。追查 Network：`/api/analyze?stock=9999` 回 **503**，body 其實是 `{"error":"抓不到 9999 的股價資料，可能是 FinMind 額度用完或代號有誤，請稍後再試"}`——後端訊息其實還算友善，但前端沒把它呈現出來，直接吞掉換成通用失敗文案。用 2317（有效代號）交叉驗證 API 本身正常（回 200，顯示「觀望」），排除是系統性 503 熔斷，確認問題是「無效代號的錯誤呈現」沒做到位。截圖 `375-查股票-9999-state1.png` |
| 持股 tab 新增一筆 → 重新整理仍在 | ✅ pass | 新增鴻海 2317（1,000 股／成本 200）後 reload，持股仍在列表（`375-持股-after-reload.png`）。同時發現：新增 2330（100 股／成本 1,000）後，因 2330 剛好在每日 tracked 清單裡，正確秀出股價 2,290、+129.0%、續抱、防守價 2,106.8；但 **2317 不在 tracked 清單，整列只顯示「—」「—」，完全沒有股價／損益／primary_decision／防守價**（`375-持股-bug-2317-no-data.png`）。Network 確認持股頁只打了 `/data/daily.json`，**從未對非 tracked 持股呼叫 `/api/analyze` 或 `/data/stocks/{id}.json`**——這代表 spec §4.2「每檔顯示 primary_decision＋防守價＋損益」對「使用者自己加的、非核心追蹤清單」的持股完全不成立，是持股 tab 的核心功能缺口，不是邊角案例（persona 就是自己手動加持股）|

## E. 資料正確性抽查

| 項目 | 結果 | 證據 |
|---|---|---|
| 頁首資料日期合理性 | 🟡 **待確認** | 2026-07-18 是**星期六**（非交易日），頁首徽章顯示「07-18 12:00 已更新」，`daily.json` 的 `data_date` 與 `generated_at` 都標 2026-07-18。字面上符合任務指示的「今天 07-18 或前一交易日」（今天＝07-18），但概念上台股週六沒開盤，這份資料實際內容應該是週五（07-17）收盤後的資料，用「07-18」標示等於用生成日而非交易日蓋日期戳，會讓人誤以為今天有更新的即時行情。不算硬性 fail，但建議 Andy 確認排程邏輯是否該標「資料對應交易日」而非「腳本執行日」|
| 大盤數字合理性 | 🔴 **fail（強烈懷疑資料源錯置）** | `daily.json` 的 `market.taiex.close = 42671.3`、`change_pct = -6.47%`。台股加權指數歷史上從未到過 4 萬點區間（近年多在 17,000–24,000 間），單日跌 6.47% 也是等同熔斷等級的崩盤，若真發生不可能只在這裡看到、其他管道毫無討論。42,671 這個數字更接近道瓊工業指數（Dow Jones，2025 年約在 42,000–45,000 區間）的量級，高度懷疑是資料源接錯（把美股道瓊數字寫進了 TAIEX 欄位，或指數代碼對應錯誤）。**這是會直接誤導 Andy 對大盤現況判讀的資料錯誤，建議列最高優先修**|
| 今日總結卡結論一致性 | ✅ pass | `偏空防禦` + `風險溫度 9/10` + 結論句「今天不加碼，守好停損位」三者方向一致、不矛盾 |
